# EventBus описание


## 1. Назначение EventBus

Централизованный механизм доставки событий между модулями. Модули не знают друг друга, общение происходит только через публикацию и подписку.

---

## 2. Требования

- O(1) регистрация подписчиков
- O(N) доставка события подписчикам
- Потокобезопасность (минимальная, только для публикации)
- Минимальные аллокации
- Отсутствие зависимостей от UI и других модулей

---

## 3. Типы событий

Событие — простая структура данных:
```cpp
struct EmailFetched { /* поля */ };
struct DocumentGenerated { /* поля */ };
```

Требования:
- без виртуальных методов
- без логики
- только данные

---

## 4. Интерфейс EventBus

### 4.1 Публикация события
```cpp
template<typename Event>
void publish(const Event& event);
```

### 4.2 Подписка
```cpp
template<typename Event>
void subscribe(std::function<void(const Event&)> handler);
```

### 4.3 Отписка (опционально)
```cpp
SubscriptionHandle subscribe(...);  
handle.unsubscribe();
```

---

## 5. Механизм работы

### 5.1 Хранение подписчиков

Для каждого типа события хранится список обработчиков:
```cpp
unordered_map<type_index, vector<Handler>>
```

Где `Handler` — обёртка над `std::function`.

### 5.2 Публикация

- Определяется тип события
- Извлекается список подписчиков
- Каждый вызывается последовательно

### 5.3 Потокобезопасность

- Мьютекс только на момент добавления/чтения списка подписчиков
- Вызов обработчиков — без блокировок

---

## 6. Ограничения

- Обработчики не должны блокировать поток
- Обработчики не должны публиковать события рекурсивно без необходимости
- EventBus не хранит состояние

---

## 7. Минимальный пример использования

### 7.1 Подписка
```cpp
bus.subscribe<EmailFetched>([](const EmailFetched& e) {
    // обработка
});
```

### 7.2 Публикация
```cpp
EmailFetched ev{/*...*/};
bus.publish(ev);
```
