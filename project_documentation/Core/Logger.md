# Описание Logger 

## 1. Назначение Logger

**Logger** — централизованный сервис логирования, используемый всеми модулями приложения. Он обеспечивает единый формат логов, запись в файл, вывод отладочной информации и фиксацию ошибок.

Logger создаётся на этапе **InitCore** и доступен всем модулям через Core.

---

## 2. Требования

- Инициализация при старте приложения
- Один глобальный экземпляр
- Потокобезопасная запись
- Минимальные аллокации
- Высокая скорость работы
- Независимость от UI и других модулей
- Поддержка уровней логирования
- Чёткий формат сообщений

---

## 3. Уровни логирования

Logger поддерживает стандартные уровни:
```cpp
enum class LogLevel {
    Debug,
    Info,
    Warning,
    Error,
    Critical
};
```

Требования:

- уровни должны быть лёгкими (enum)
- без виртуальных методов
- без логики внутри уровней

---

## 4. Интерфейс Logger

### 4.1 Запись сообщения
```cpp
void log(LogLevel level, const std::string& message);
```

### 4.2 Упрощённые методы
```cpp
void debug(const std::string& msg);
void info(const std::string& msg);
void warn(const std::string& msg);
void error(const std::string& msg);
void critical(const std::string& msg);
```

### 4.3 Установка уровня логирования
```cpp
void setLevel(LogLevel level);
```

### 4.4 Получение текущего уровня
```cpp
LogLevel level() const;
```

---

## 5. Механизм работы

### 5.1 Хранение состояния

Logger хранит:
```cpp
std::ofstream file;
LogLevel currentLevel;
std::mutex mutex;
```

### 5.2 Форматирование сообщения

Каждое сообщение имеет формат:
```
[2026-06-09 15:42:11] [INFO] ModuleName: текст сообщения
```

Компоненты:
- timestamp
- уровень
- имя модуля
- текст

### 5.3 Потокобезопасность

- запись в файл защищена мьютексом
- форматирование выполняется локально в стеке
- чтение уровня — без блокировок

### 5.4 Производительность

- файл открыт один раз
- запись буферизована
- минимум выделений памяти

---

## 6. Ограничения

- Logger не должен зависеть от UI
- Logger не должен зависеть от модулей
- Logger не выполняет бизнес‑логику
- Logger не должен использовать тяжёлые операции (JSON, regex)
- Logger не должен менять состояние приложения

---

## 7. Минимальный пример использования

### 7.1 Простое сообщение
```cpp
logger.info("Application started");
```

### 7.2 Ошибка
```cpp
logger.error("Failed to connect to database");
```

### 7.3 Отладка
```cpp
logger.debug("Email fetched: " + email.id);
```

### 7.4 Установка уровня
```cpp
logger.setLevel(LogLevel::Warning);
```
