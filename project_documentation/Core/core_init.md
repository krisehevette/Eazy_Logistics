# Инициализация Core‑модуля


## 1. Назначение этапа

Этап **InitCore** создаёт фундаментальные сервисы, на которых держится всё приложение. Он не загружает модули и не взаимодействует с UI — только подготавливает ядро.

---

## 2. Место в жизненном цикле

Этап вызывается из `Application::Run()`:
```C++
Startup::Run();
InitCore::Run();   //← вот этот этап
InitModules::Run();
InitUI::Run();
MainLoop::Run();
Shutdown::Run();
```

---

## 3. Последовательность инициализации

### 3.1 Создание EventBus

- Создаётся глобальный экземпляр EventBus
- Подготавливается хранилище подписчиков
- Гарантируется потокобезопасность для публикации
- EventBus ещё пуст — подписки появятся позже, на этапе InitModules

### 3.2 Загрузка Config

- Чтение конфигурационных файлов
- Валидация путей, проверка наличия ресурсов
- Подготовка структуры настроек для всех модулей

Config не зависит от модулей — только от файловой системы.

### 3.3 Инициализация Logger

- Создание логгера
- Открытие лог‑файла
- Настройка уровней логирования
- Привязка к глобальному контексту приложения

Logger начинает работать сразу, чтобы фиксировать дальнейшие этапы.

### 3.4 Создание ApplicationState

- Создаётся структура глобального состояния
- Пока пустая: модули ещё не загружены
- Содержит данные, которые будут обновляться событиями

### 3.5 Создание ModuleRegistry

- Подготавливается реестр модулей
- Он ещё не содержит модулей — только пустую структуру
- Позже сюда будут добавляться контракты (InitModules)

---

## 4. Принципы, которые соблюдаются на этапе InitCore

- **Модульность** — ни один модуль ещё не создан
- **Событийная модель** — EventBus готов, но пуст
- **SOLID** — каждый компонент Core независим
- **Никаких зависимостей от UI**
- **Никакой бизнес‑логики**

---
