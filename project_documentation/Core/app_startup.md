# Запуск приложения


## 1. Цель документа

Определить точку входа приложения и описать базовые действия, которые выполняются операционной системой и программой до начала инициализации Core‑модуля.

Документ является частью общей схемы инициализации приложения и описывает **Этап 1 — Запуск процесса**.

---

## 2. Структура кода запуска приложения

Раздел описывает только структуру кода, отвечающего за запуск процесса и передачу управления системе инициализации. Подробные этапы инициализации описаны в отдельных документах.

Запуск приложения реализован в виде цепочки вызовов, начинающейся в `main.cpp` и продолжающейся через модуль `Core/Application`.

### 2.1 main.cpp

Файл `main.cpp` содержит только точку входа приложения и не включает
никакой логики инициализации:

```cpp
#include "Core/Application.h"

int WINAPI WinMain(HINSTANCE hInst, HINSTANCE, LPSTR, int)
{
    return Application::Run();
}
```

Его единственная задача — передать управление классу `Application`.

### 2.2 Application.h / Application.cpp

Файлы:

```
src/Core/Application.h
src/Core/Application.cpp
```

Класс `Application` является **центральным оркестратором инициализации**. Он определяет последовательность этапов запуска:

- Startup
- InitCore
- InitModules
- InitUI
- MainLoop
- Shutdown

Пример структуры:

```cpp
int Application::Run()
{
    Startup::Run();
    InitCore::Run();
    InitModules::Run();
    InitUI::Run();
    MainLoop::Run();
    Shutdown::Run();
    return 0;
}
```

### 2.3 Этапы запуска инициализации (Initialization/)

Каждый этап вынесен в отдельный файл:

```
src/Core/Initialization/
  Startup.h / Startup.cpp
  InitCore.h / InitCore.cpp
  InitModules.h / InitModules.cpp
  InitUI.h / InitUI.cpp
```

Назначение:

- **Startup** — базовая подготовка среды (обработчики ошибок, пути, проверка файлов).
- **InitCore** — создание EventBus, Config, Logger, ApplicationState, ModuleRegistry.
- **InitModules** — регистрация и инициализация модулей (Database, Email, AI, Logistics, Documents).
- **InitUI** — создание окна, инициализация DirectX, загрузка шрифтов, построение UI‑дерева.

Каждый этап полностью изолирован и отвечает только за свою часть запуска.

### 2.4 Главный цикл (MainLoop/)

```
src/Core/MainLoop/
  MainLoop.h
  MainLoop.cpp
```

Содержит основной цикл приложения:

- обработка системных сообщений,
- обновление состояния,
- вызов рендера UI.

### 2.5 Завершение работы (Shutdown/)

```
src/Core/Shutdown/
  Shutdown.h
  Shutdown.cpp
```

Отвечает за:

- остановку фоновых задач,
- сохранение состояния,
- корректное завершение модулей,
- освобождение ресурсов.
