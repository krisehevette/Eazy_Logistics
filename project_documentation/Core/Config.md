# Config описание

## 1. Назначение Config

Централизованный сервис загрузки и хранения настроек приложения. Config предоставляет единый источник правды для всех модулей и гарантирует, что параметры окружения, пути, ключи и настройки модулей доступны в безопасной и неизменяемой форме.

Модули **не читают файлы напрямую** — только через Config.

---

## 2. Требования

- Загрузка настроек при старте приложения (этап InitCore)
- Только один экземпляр (singleton‑подобная модель через Core)
- Отсутствие зависимостей от UI и других модулей
- Минимальные аллокации
- Быстрый доступ к параметрам (O(1) по ключу)
- Поддержка вложенных секций (например: `email.server.host`)
- Валидация обязательных параметров
- Никакой логики, кроме чтения и хранения данных

---

## 3. Формат настроек

Config загружает данные из одного или нескольких файлов:

- `config.json`
- `config.local.json` (опционально)
- `config.dev.json` (опционально)

Структура — иерархическая:
```json
{
    "paths": {
        "logs": "logs/app.log",
        "database": "data/db.sqlite"
    },
    "email": {
        "server": "imap.gmail.com",
        "port": 993
    }
}
```

Требования к данным:
- только примитивы (string, int, bool, float)
- вложенные объекты
- без логики
- без ссылок на другие модули

---

## 4. Интерфейс Config

### 4.1 Загрузка настроек
```cpp
void load(const std::string& path);
```

### 4.2 Получение значения по ключу
```cpp
template<typename T>
T get(const std::string& key) const;
```

### 4.3 Проверка наличия ключа
```cpp
bool has(const std::string& key) const;
```

### 4.4 Доступ к секции
```cpp
ConfigSection section(const std::string& prefix) const;
```

---

## 5. Механизм работы

### 5.1 Хранение данных

Все параметры хранятся в виде плоской таблицы:
```cpp
unordered_map<string, ConfigValue>
```

Где ключ — строка вида:
```cpp
"email.server"
"paths.logs"
"database.connection.timeout"
```

Это обеспечивает O(1) доступ.

### 5.2 Загрузка

- Чтение JSON‑файла
- Рекурсивное разворачивание вложенных объектов в плоские ключи
- Валидация обязательных параметров
- Объединение нескольких файлов (local → override)

### 5.3 Потокобезопасность

- Config создаётся один раз при InitCore
- После загрузки — **только чтение**
- Мьютексы не требуются

---

## 6. Ограничения

- Config не изменяется во время работы приложения
- Config не выполняет логику модулей
- Config не хранит сложные структуры (классы, массивы объектов)
- Config не зависит от EventBus, UI, Database и других модулей

---

## 7. Минимальный пример использования

### 7.1 Получение строки
```cpp
std::string host = config.get<std::string>("email.server");
```

### 7.2 Получение числа
```cpp
int port = config.get<int>("email.port");
```

### 7.3 Проверка ключа
```cpp
if (config.has("paths.logs")) {
    auto path = config.get<std::string>("paths.logs");
}
```

### 7.4 Работа с секцией
```cpp
auto email = config.section("email");
auto host = email.get<std::string>("server");
```
