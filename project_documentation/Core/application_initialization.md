# Инициализация приложения  


## 1. Цель документа

Определить общую структуру и последовательность этапов инициализации приложения. Документ служит точкой входа для понимания того, как система подготавливает ядро, модули и интерфейс перед переходом в основной цикл работы.

Подробное описание каждого этапа находится в отдельных документах

---

## 2. Общая схема инициализации

Инициализация приложения состоит из следующих основных этапов:

1. [**Запуск процесса**](/project_documentation/Core/app_startup.md)
   Создание точки входа, базовая подготовка среды.

2. [**Инициализация Core‑модуля**](/project_documentation/Core/core_init.md)
   Создание EventBus, загрузка конфигурации, инициализация логирования, создание глобального состояния, подготовка ModuleRegistry.

3. **Регистрация модулей**  
   Каждый модуль объявляет свои контракты, события и подписки.

4. **Инициализация модулей**  
   Запуск внутренней логики модулей: БД, почта, AI, логистика, документы.

5. **Инициализация UI**  
   Создание окна, инициализация DirectX, загрузка шрифтов, построение UI‑дерева.

6. **Публикация события `ApplicationStarted`**  
   Система переходит в рабочее состояние, запускаются фоновые процессы.

7. **Переход в главный цикл приложения**  
   Управление событиями, обновление состояния, вызов рендера UI.
