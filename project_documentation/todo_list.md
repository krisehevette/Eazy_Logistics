# Список задач

- Определить общую форму контрактов модулей.
- Описать Data Flow между модулями.
- Подготовить минимальный Core (EventBus, Logger, AppState).

---


## Задачи на сегодня

### 1. Определить структуру UI‑модуля

- **UI Core** — базовые классы: Widget, Layout, Style.
- **UI Renderer** — DirectX‑часть.
- **UI State** — фокус, hover, active, capture.
- **UI Events** — мышь, клавиатура, жесты.
- **UI Theme Engine** — цвета, тени, скругления.
- **UI Animation Engine** — таймлайны, easing.

### 2. Спроектировать дизайн‑систему

- **Цветовая палитра** — светлая/тёмная тема, акценты.
- **Типографика** — размеры, веса, иерархия.
- **Компоненты** — кнопки, поля, таблицы, панели.
- **Iconography** — стиль иконок, размеры.

### 3. Спроектировать структуру экранов приложения

- **Главный Dashboard**
- **Модуль Email**
- **Модуль Logistics**
- **Модуль Documents**
- **Модуль Database**

Каждый экран — отдельный layout + набор виджетов.

### 4. Определить взаимодействие UI ↔ EventBus

- **UI слушает события** — обновление данных.
- **UI публикует события** — действия пользователя.
- **UI не знает бизнес‑логики** — только отображение.
