# Eazy Logistics — Project Context

## 1. Название и краткое описание проекта

**Eazy Logistics** — настольное приложение на **C++**, предназначенное для автоматизации и оптимизации работы компании‑посредника в сфере грузоперевозок.

Система объединяет:

- управление клиентами и подрядчиками,
- документооборот,
- анализ электронной почты локальными LLM,
- отслеживание грузов,
- современный UI,
- единое хранилище данных,
- автоматизацию подготовки документов.

Цель — создать **быструю, надёжную, красивую и умную** платформу, которая заменяет десятки разрозненных инструментов.

---

## 2. Основные цели проекта

- Централизованная база клиентов, подрядчиков, грузов и документов.
- Интуитивный, современный, визуально привлекательный UI.
- Автоматизация рутинных процессов (документы, письма, статусы).
- Интеграция локальных LLM для анализа почты и генерации документов.
- Отслеживание грузов в реальном времени.
- Минимизация ручной работы сотрудников.
- Высокая скорость работы и отказоустойчивость.
- Возможность масштабирования и расширения модулей.

> ---
>
> ## 5. Модель взаимодействия
>
> ### 5.1 EventBus
>
> Центральный механизм обмена сообщениями. Модуль может:
>
> - **опубликовать событие**
> - **подписаться на событие**
>
> EventBus не знает, кто публикует и кто слушает.
>
> ### 5.2 События
>
> Событие — это **структура данных**, описывающая факт, например:
>
> - `EmailFetched`
> - `EmailClassified`
> - `CargoUpdated`
> - `DocumentGenerated`
>
> События не содержат логики.
>
> ### 5.3 Контракты модулей
>
> Контракт включает:
>
> - интерфейсы (например `IEmailClient`, `IDocumentGenerator`)
> - список событий, которые модуль **генерирует**
> - список событий, которые модуль **слушает**
>
> Контракт — это “официальный язык общения” модуля с системой.
>
> ---
>
> ## 6. Потоки данных (пример)
>
> 1. Email → публикует `EmailFetched`
> 2. AI → слушает `EmailFetched`, публикует `EmailClassified`
> 3. Database → слушает `EmailClassified`, сохраняет данные
> 4. UI → слушает `EmailClassified`, обновляет интерфейс
> 5. Documents → слушает `EmailClassified`, предлагает шаблоны
>
> Модули не знают друг друга — только события.
>
> ---
>
> ## 7. Расширяемость
>
> Чтобы добавить новый модуль:
>
> 1. Создать папку.
> 2. Определить контракт (интерфейсы + события).
> 3. Зарегистрировать подписки в Core.
> 4. Реализовать модуль.
>
> Существующие модули менять не требуется.

## 4. Принципы разработки

- Ведение документации, концентрация на "фундаменте" приложения, отсутствие спешки, последовательность.
- Безопасность данных от утечек.
- Полная инкапсуляция, модули независимы.
- Соблюдение парадигм ООП и SOLID.
- Максимальная скорость работы программы даже на слабом железе.
- Передовой, интуитивный, быстрый интерфейс.
- Лёгкая тестируемость.
- Возможность работы с кодом для сторонних / наемных разработчиков.
